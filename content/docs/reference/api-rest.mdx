## Overview

The grith API server runs on `localhost:3141` by default and provides programmatic access to the proxy, digest queue, audit logs, configuration, and supervisor. All endpoints return JSON. The API is designed for local use and integration with the web dashboard.

<Terminal command="grith dashboard start" />

## Authentication

The API binds to `127.0.0.1` by default and does not require authentication for local access. If you expose the API via a reverse proxy, you must add your own authentication layer. See the [Reverse Proxy guide](/docs/guides/reverse-proxy).

## Error format

All error responses follow a consistent shape:

<CodeBlock language="json" code={`{
  "error": "Human-readable error message",
  "code": "ERROR_CODE"
}`} />

Feature-gated endpoints (Pro or Enterprise) return HTTP 403 with code `FEATURE_GATED` when the required plan is not active:

<CodeBlock language="json" code={`{
  "error": "This feature requires a Pro subscription",
  "code": "FEATURE_GATED",
  "current_tier": "community",
  "required_tier": "pro",
  "upgrade_url": "https://grith.ai/pricing"
}`} />

## Digest

The digest queue holds tool calls that scored in the QUEUE zone. These endpoints let you review and act on pending items.

### GET /digest

List pending digest items.

**Query parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `limit` | Integer | 50 | Maximum number of items to return |
| `offset` | Integer | 0 | Pagination offset |

**Response:**

<CodeBlock language="json" code={`{
  "items": [
    {
      "id": "d_abc123",
      "tool": "write_file",
      "args": {"path": "/home/user/.bashrc", "content": "..."},
      "score": 4.5,
      "filters": [
        {"name": "path_match", "score": 2.0},
        {"name": "operation_risk", "score": 2.0},
        {"name": "allowlist", "score": 0.5}
      ],
      "created_at": "2026-02-24T10:30:00Z",
      "status": "pending"
    }
  ],
  "total": 1,
  "limit": 50,
  "offset": 0
}`} />

### POST /digest/:id/approve

Approve a queued tool call. The call will be executed.

**Response:** `{"status": "approved"}`

### POST /digest/:id/deny

Deny a queued tool call. The call is blocked and logged.

**Response:** `{"status": "denied"}`

### POST /digest/:id/learn

Submit feedback for adaptive learning. Provide a signal indicating whether the score was too high (false positive) or too low (false negative).

**Request body:**

<CodeBlock language="json" code={`{
  "signal": "false_positive"
}`} />

Valid signals: `false_positive`, `false_negative`, `correct`.

**Response:** `{"status": "learned"}`

### POST /digest/:id/escalate

Escalate a digest item to a higher-severity notification channel.

**Response:** `{"status": "escalated"}`

### POST /digest/:id/unlock-egress

Approve an outbound network request that was blocked by the egress policy filter. This is a targeted approval that allows only the specific blocked request.

**Response:** `{"status": "egress_unlocked"}`

### POST /digest/:id/deny-terminate

Deny the tool call and terminate the session that produced it. Use this for clearly malicious activity.

**Response:** `{"status": "denied_terminated"}`

### POST /digest/:id/allow-always

Approve the tool call and add a permanent allowlist rule so similar calls are auto-allowed in the future.

**Response:** `{"status": "allowed_always"}`

### POST /digest/:id/webhook-review

Send the digest item to a configured webhook for external review (e.g., a Slack approval workflow).

**Response:** `{"status": "webhook_sent"}`

## Audit

### GET /audit

Query audit log entries.

**Query parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `limit` | Integer | 100 | Maximum entries |
| `offset` | Integer | 0 | Pagination offset |
| `session` | String | — | Filter by session ID |
| `decision` | String | — | Filter by decision: `allow`, `queue`, `deny` |
| `from` | ISO 8601 | — | Start time |
| `to` | ISO 8601 | — | End time |

**Response:**

<CodeBlock language="json" code={`{
  "entries": [
    {
      "id": "a_xyz789",
      "session_id": "s_abc123",
      "tool": "execute_command",
      "args": {"command": "ls -la"},
      "composite_score": 1.2,
      "decision": "allow",
      "filters": [...],
      "timestamp": "2026-02-24T10:00:00Z"
    }
  ],
  "total": 42,
  "limit": 100,
  "offset": 0
}`} />

### GET /audit/:id

Get a single audit entry by ID.

**Response:** Single audit entry object (same shape as above).

### GET /audit/export

Export audit logs in bulk.

**Query parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `format` | String | `"json"` | Export format: `json`, `csv` |
| `from` | ISO 8601 | — | Start time |
| `to` | ISO 8601 | — | End time |

**Response:** JSON array or CSV file depending on format.

### GET /audit/exfil-stats

Get exfiltration detection statistics.

**Response:**

<CodeBlock language="json" code={`{
  "canary_triggers": 3,
  "dlp_blocks": 12,
  "egress_denials": 7,
  "taint_propagations": 45,
  "period": "7d"
}`} />

## Config

### GET /config

Retrieve the current resolved configuration.

**Response:** Full configuration object in JSON format.

### PUT /config

Update configuration values. Accepts a partial configuration object. Only provided keys are updated.

**Request body:**

<CodeBlock language="json" code={`{
  "proxy": {
    "auto_allow_threshold": 3.0
  }
}`} />

**Response:** `{"status": "updated"}`

<Callout type="warning" title="Runtime updates">
Configuration changes via the API take effect immediately without a restart. Be cautious when adjusting thresholds on a running system.
</Callout>

## Server

### POST /server/shutdown

Gracefully shut down the grith server. Active sessions are terminated and the digest queue is flushed to disk.

**Response:** `{"status": "shutting_down"}`

<Callout type="danger" title="Destructive operation">
This endpoint terminates the server process. All active supervisor sessions will be stopped. The digest queue is persisted before shutdown.
</Callout>

## Supervisor

### GET /supervisor/sessions

List all active supervisor sessions.

**Response:**

<CodeBlock language="json" code={`{
  "sessions": [
    {
      "id": "sup_abc123",
      "pid": 12345,
      "profile": "claude-code",

Create a new supervisor session programmatically.

**Request body:**

<CodeBlock language="json" code={`{

  "args": [],
  "profile": "claude-code"

### DELETE /supervisor/sessions/:id
