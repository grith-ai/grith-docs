## Overview

The `grith` CLI is the primary interface for interacting with the grith security proxy, supervisor, and audit system. Every command supports the global flags listed below, and most commands can be composed with shell pipelines.

<Terminal command="grith --help" />

## Global flags

These flags are available on every command and subcommand.

| Flag | Type | Description |
|---|---|---|
| `--config <CONFIG>` | Path | Path to config file (default: `~/.config/grith/config.toml`) |
| `--log-level <LOG_LEVEL>` | Enum | Logging verbosity: `trace`, `debug`, `info`, `warn`, `error` |
| `--no-color` | Boolean | Disable ANSI colour output |
| `--project <PROJECT>` | String | Override the active project context |

## Commands

### grith

Start the grith REPL and API server on `:3141`. This is the default entry point when no subcommand is given. The REPL provides an interactive session where you can issue tasks, review digest items, and inspect proxy state.

<Terminal command="grith" />

---

### grith run

Execute a single task non-interactively. This is the primary entry point for running AI agent tasks through the grith security proxy.

<Terminal command="grith run &quot;refactor the auth module to use JWT&quot;" />

The task string is passed to the configured LLM provider. All tool calls made during execution are routed through the scoring proxy, and any calls that land in the QUEUE zone are held for digest review.

| Argument | Required | Description |
|---|---|---|
| `<TASK>` | Yes | Natural language task description |

---

### grith init

Create a default configuration file at `~/.config/grith/config.toml`. If a configuration file already exists, `init` will not overwrite it unless `--force` is passed.

<Terminal command="grith init" />

---

### grith config

Show or modify the active configuration.

<Terminal command="grith config" />

Displays the fully resolved configuration, including defaults and environment variable overrides.

#### grith config set

Set a specific configuration key.

<Terminal command="grith config set proxy.auto_allow_threshold 3.0" />

Changes are written to the config file and take effect on the next proxy restart.

---

### grith audit

Browse audit logs interactively. Every tool call, score result, and user decision is recorded in the audit log.

<Terminal command="grith audit" />

#### grith audit export

Export audit logs in structured format.

<Terminal command="grith audit export --format json --output audit.json" />

Supported formats: `json`, `csv`.

---

### grith digest

Manage the digest queue. Tool calls that score in the QUEUE zone (between the allow and deny thresholds) are held here for human review.

<Terminal command="grith digest" />

#### grith digest review

Launch an interactive review session for pending digest items.

<Terminal command="grith digest review" />

During review, each queued tool call is presented with its score breakdown. You can approve, deny, or provide feedback that feeds into the adaptive learning system.

---

### grith canary

Manage canary tokens for exfiltration trap detection. Canary tokens are planted strings that, if they appear in outbound tool calls, indicate data exfiltration.

#### grith canary list

<Terminal command="grith canary list" />

List all active canary tokens.

#### grith canary add

<Terminal command="grith canary add --label &quot;database credentials&quot;" />

Generate and register a new canary token.

#### grith canary remove

<Terminal command="grith canary remove <TOKEN_ID>" />

Remove a canary token by its ID.

#### grith canary rotate

<Terminal command="grith canary rotate <TOKEN_ID>" />

Rotate a canary token, generating a new value while preserving the label and detection history.

---

### grith proxy

Interact with the scoring proxy directly.

#### grith proxy test

Dry-run a tool call against the proxy scoring pipeline without executing it. Useful for tuning filter thresholds.

<Terminal command='grith proxy test &apos;{"tool": "write_file", "args": {"path": "/etc/passwd", "content": "test"}}&apos;' />

Exit codes carry semantic meaning:

| Exit code | Meaning |
|---|---|
| `0` | ALLOW — score below allow threshold |
| `1` | QUEUE — score in review zone |
| `2` | DENY — score above deny threshold |

<Callout type="tip" title="Filter tuning workflow">
Use `grith proxy test` in a loop to validate your filter configuration before deploying changes. Combine with `--log-level debug` to see per-filter score contributions.
</Callout>

### grith exec

Supervise an external CLI tool with OS-level syscall interception. This is grith's supervisor mode, which uses ptrace and seccomp-BPF on Linux to intercept and evaluate system calls made by the supervised process.

<Terminal command="grith exec --profile claude-code -- claude" />

| Flag | Description |
|---|---|
| `--profile <PROFILE>` | Supervisor profile to apply (e.g., `claude-code`, `codex`, `aider`, `generic`) |
| `--attach <PID>` | Attach to an already-running process instead of spawning a new one |

Everything after `--` is the command to supervise. The supervised process runs in a sandboxed environment where file access, network calls, and process creation are evaluated against the active profile.

<Callout type="warning" title="Linux only">
Full syscall interception requires Linux with ptrace and seccomp-BPF support. On macOS, grith falls back to process-lifecycle monitoring, which provides less granular control.
</Callout>

### grith supervisor

List and manage active supervisor sessions.

#### grith supervisor list

<Terminal command="grith supervisor list" />

Show all active supervisor sessions with their PIDs, profiles, and uptime.

#### grith supervisor status

<Terminal command="grith supervisor status <SESSION_ID>" />

Show detailed status for a specific supervisor session, including intercepted syscall counts and active alerts.

#### grith supervisor kill

<Terminal command="grith supervisor kill <SESSION_ID>" />

Terminate a supervised process and its session.

### grith dashboard

#### grith dashboard start

<Terminal command="grith dashboard start" />

#### grith dashboard stop

<Terminal command="grith dashboard stop" />

Gracefully shut down the dashboard server.

#### grith dashboard status

<Terminal command="grith dashboard status" />

Check whether the dashboard is running and show its URL.

### grith log

View audit-backed session logs.

<Terminal command="grith log" />

| Flag | Description |
|---|---|
| `--tail` | Stream new log entries in real time |
| `--session <NAME\|UUID>` | Filter logs to a specific session by name or UUID |
| `--limit <N>` | Maximum number of entries to display |

<Terminal command="grith log --tail --session abc123" />

## Exit codes

All grith commands follow a consistent exit code convention:

| Exit code | Meaning |
|---|---|
| `0` | Success |
| `1` | General error or QUEUE result (for `proxy test`) |
| `2` | DENY result (for `proxy test`) |

## Environment variables

Configuration values can also be set via environment variables. The naming convention is `GRITH_` followed by the TOML path with underscores:

<Terminal command="GRITH_PROXY_AUTO_ALLOW_THRESHOLD=3.0 grith run &quot;my task&quot;" />

See [Configuration](/docs/reference/config) for the full list of configuration options.
