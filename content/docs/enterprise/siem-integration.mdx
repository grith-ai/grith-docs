SIEM and SOAR integration is planned for grith v2.0. This page describes the planned export formats, platform integrations, and configuration model.

grith evaluates every tool call from AI agents against 17 security filters and records every decision in a tamper-evident audit log. SIEM integration lets you stream those events into your existing security operations platform so that AI agent activity appears alongside your other security telemetry — same dashboards, same alerting rules, same incident workflows.

## Export formats

### CEF (Common Event Format)

CEF is the most widely supported SIEM ingestion format. grith maps its event fields to CEF as follows:

```
CEF:0|grith|grith-proxy|1.0|tool_call_denied|Tool Call Denied|8|
  src=127.0.0.1 suser=alice@example.com
  cs1Label=tool cs1=write_file
  cs2Label=score cs2=8.7
  cs3Label=decision cs3=DENY
  cs4Label=filters cs4=path_traversal,sensitive_path,write_outside_project
  cs5Label=session cs5=abc123
  rt=Feb 24 2026 14:30:00
```

### LEEF (Log Event Extended Format)

For IBM QRadar environments:

```
LEEF:2.0|grith|grith-proxy|1.0|tool_call_denied|
  src=127.0.0.1 usrName=alice@example.com
  tool=write_file score=8.7 decision=DENY
  filters=path_traversal,sensitive_path,write_outside_project
  sessionId=abc123
```

## Platform configuration

### Splunk

<ConfigExample code={`[enterprise.siem]
enabled = true
platform = "splunk"
format = "json"

[enterprise.siem.splunk]
hec_url = "https://splunk.internal.example.com:8088"
hec_token_env = "GRITH_SPLUNK_HEC_TOKEN"
index = "grith_security"
source = "grith-proxy"
sourcetype = "grith:event"
verify_tls = true
`} title="Splunk HEC configuration" />

### Elastic SIEM

<ConfigExample code={`[enterprise.siem]

### Generic syslog

For any SIEM that accepts syslog input:

<ConfigExample code={`[enterprise.siem]
enabled = true
platform = "syslog"
format = "cef"

[enterprise.siem.syslog]
host = "syslog.internal.example.com"
port = 6514
protocol = "tls"                  # tcp, udp, tls
facility = "local0"
severity_mapping = true           # Map DENY → alert, QUEUE → warning, ALLOW → info
`} title="Generic syslog configuration" />

## OpenTelemetry export

For organizations using observability platforms (Datadog, Grafana, Honeycomb, New Relic), grith will export events and metrics via the OpenTelemetry protocol:

<ConfigExample code={`[enterprise.otel]
enabled = true

[enterprise.otel.traces]
endpoint = "https://otel-collector.internal.example.com:4317"
protocol = "grpc"                 # grpc or http

[enterprise.otel.metrics]
endpoint = "https://otel-collector.internal.example.com:4317"
protocol = "grpc"
export_interval = "30s"

[enterprise.otel.resource]
service_name = "grith-proxy"
environment = "production"
`} title="OpenTelemetry configuration" />

### Exported metrics

| Metric | Type | Description |
|---|---|---|
| `grith.proxy.evaluations.total` | Counter | Total tool calls evaluated |
| `grith.proxy.decisions.allow` | Counter | Tool calls auto-allowed |
| `grith.proxy.decisions.queue` | Counter | Tool calls quarantined |
| `grith.proxy.decisions.deny` | Counter | Tool calls blocked |
| `grith.proxy.score.histogram` | Histogram | Distribution of composite scores |
| `grith.proxy.evaluation.duration` | Histogram | Time to evaluate a tool call (ms) |
| `grith.supervisor.sessions.active` | Gauge | Currently active supervisor sessions |
| `grith.canary.triggers.total` | Counter | Canary token trigger events |

## Webhook-based forwarding

For custom integrations or platforms not listed above, grith will support HTTPS webhook delivery:

<ConfigExample code={`[enterprise.siem]
enabled = true
platform = "webhook"
format = "json"

[enterprise.siem.webhook]
url = "https://your-endpoint.example.com/grith/events"
auth_header_env = "GRITH_WEBHOOK_AUTH_HEADER"
batch_size = 50
flush_interval = "10s"
retry_attempts = 3
retry_backoff = "exponential"
`} title="Webhook configuration" />

Webhook payloads are delivered as JSON arrays of events, batched for efficiency. Each delivery includes an HMAC-SHA256 signature in the `X-Grith-Signature` header for verification.

## Real-time event streaming

For use cases that require sub-second event delivery, grith will support WebSocket streaming:

<ConfigExample code={`[enterprise.siem.streaming]
enabled = true
bind_address = "127.0.0.1:3142"
auth_required = true
max_connections = 10
`} title="WebSocket streaming configuration" />

Connect to the stream:

```
ws://127.0.0.1:3142/events?token=<session_token>
```

Events are delivered as individual JSON objects over the WebSocket connection. This is useful for building custom dashboards, real-time alerting pipelines, or integration with SOAR platforms.

## Event filtering

Not every event needs to reach your SIEM. You can filter which events are exported based on decision, score, or event type:

<ConfigExample code={`[enterprise.siem.filters]
min_score = 3.0                   # Only export events scoring 3.0+
decisions = ["QUEUE", "DENY"]     # Only export quarantined and blocked calls
event_types = [                   # Specific event types to export
  "tool_call_evaluated",
  "canary_triggered",
  "rbac_access_denied",
  "policy_changed",
  "audit_chain_broken",
]
`} title="Event filtering" />

<Callout type="tip" title="Cost management">
SIEM ingestion costs scale with volume. Filtering to QUEUE and DENY events only can reduce ingestion volume by 80-90% while still capturing the events that matter most to your security operations team.
</Callout>
