## Overview

Every tool call processed by grith's built-in agent (`grith run`) generates an audit record that includes LLM usage metadata: which provider handled the request, which model was used, how many tokens were consumed, and the estimated cost. grith Pro aggregates this data into cost analytics accessible via the dashboard and the REST API.

## How cost tracking works

When grith routes a prompt to an LLM provider, the response includes token counts. grith records these alongside the audit entry:

| Field | Source | Description |
|---|---|---|
| `llm_provider` | Provider adapter | The provider that handled the call (e.g., `anthropic`, `openai`, `openrouter`, `ollama`) |
| `llm_model` | Provider response | The specific model used (e.g., `claude-sonnet-4-20250514`, `gpt-4o`) |
| `prompt_tokens` | Provider response | Number of tokens in the prompt |
| `completion_tokens` | Provider response | Number of tokens in the completion |
| `estimated_cost_usd` | Calculated | Estimated cost based on per-model pricing tables |

Cost estimation uses built-in pricing tables that are updated with each grith release. For custom or self-hosted models (e.g., Ollama), the estimated cost is reported as `0.00` unless you configure custom pricing.

<ConfigExample
  title="Custom model pricing"
  code={`[llm.pricing.custom]
"my-org/llama-70b" = { prompt_per_1k = 0.001, completion_per_1k = 0.002 }
"my-org/codestral" = { prompt_per_1k = 0.0005, completion_per_1k = 0.001 }`}
/>

## API endpoints

Returns a cost breakdown by provider and model for a given time period.

  code={`GET /api/analytics/cost?period=30d

| Parameter | Type | Default | Description |

| `period` | String | `"30d"` | Time period: `7d`, `30d`, `90d` |

## Dashboard

The local dashboard at `localhost:3141` displays cost data in the Analytics section. The cost view shows:

- Total spend for the current billing period
- Cost trend chart (daily granularity)
- Breakdown by provider and model
- Cost per session (for identifying expensive tasks)

<Callout type="info" title="Dashboard auto-refresh">
The dashboard polls `/api/analytics/cost` every 60 seconds. Cost data updates in near real-time as sessions run.
</Callout>

## Cloud sync

<Badge variant="pro">Pro</Badge>

When audit upload is enabled, cost data is included in the sync payload sent to grith.ai. The team dashboard aggregates cost data across all team members, providing:

- Team-wide cost by provider and model
- Per-developer cost breakdown (anonymised by default)
- Cost trends and anomaly detection

<Terminal command="grith pro sync" />

<CodeBlock
  code={`[grith] Connecting to grith.ai...
[grith] Uploading audit summary (opt-in)... done (cost data: $47.82 over 30d)
[grith] Sync complete.`}
  language="text"
/>

Cost data in the cloud is aggregated. Individual prompts, completions, and API keys are never uploaded.

<ConfigExample
  title="Enable cost sync"
  code={`[pro.sync]
upload_audit = true         # cost data included in audit summary`}
/>
