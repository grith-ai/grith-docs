## Overview

<Badge variant="enterprise">Enterprise</Badge>

grith's core functionality requires zero internet access. The security proxy, filter chain, syscall interception, audit logging, and digest system all run entirely on the local machine. This makes grith suitable for air-gapped environments where network connectivity to external services is prohibited — classified networks, SCIF environments, regulated industries, and isolated development labs.

This guide covers installing grith without internet, running local LLMs, activating an enterprise licence offline, exporting audit data for external compliance systems, and managing updates.

## Architecture in an air-gapped environment

In a standard deployment, grith optionally connects to:

| Service | Purpose | Required? |
|---|---|---|
| LLM providers (OpenAI, Anthropic, etc.) | Built-in agent intelligence | No — use local LLM |
| grith.ai | Pro/Enterprise licence, team sync | No — use offline licence |
| OS package repos | Installation and updates | No — use offline install |

In an air-gapped deployment, all three are replaced with local alternatives. The security proxy, filter chain, and supervisor work identically regardless of connectivity.

## Offline installation

### Transfer the binary

On a connected machine, download the grith release for your target platform:

<Terminal command="curl -fsSL https://github.com/grith-ai/grith/releases/download/v1.0.0/grith-linux-x86_64.tar.gz -o grith-linux-x86_64.tar.gz" />

Transfer the archive to the air-gapped machine via approved media (USB, optical disc, cross-domain solution). Then install:

<CodeBlock
  code={`# On the air-gapped machine
tar xzf grith-linux-x86_64.tar.gz
sudo cp grith /usr/local/bin/
grith --version`}
  language="bash"
/>

### Build from source

If your environment permits building from source, transfer the grith source tree and a vendored Rust toolchain:

<CodeBlock
  code={`# On the connected machine: vendor dependencies
git clone https://github.com/grith-ai/grith
cd grith
cargo vendor
tar czf grith-source-vendored.tar.gz .

# Transfer grith-source-vendored.tar.gz to air-gapped machine

# On the air-gapped machine
tar xzf grith-source-vendored.tar.gz
cd grith
cargo build --release --frozen
sudo cp target/release/grith /usr/local/bin/`}
  language="bash"
/>

### Initialise

<Terminal command="grith init" />

## Local LLM via Ollama

grith's built-in agent needs an LLM. In air-gapped environments, use Ollama to run models locally.

### Install Ollama offline

On a connected machine, download the Ollama binary and one or more models:

<CodeBlock
  code={`# On connected machine
curl -fsSL https://ollama.com/download/ollama-linux-amd64 -o ollama
chmod +x ollama

# Pull models while connected
./ollama pull llama3.1:8b
./ollama pull codellama:13b

# Package the model blobs
tar czf ollama-models.tar.gz ~/.ollama/models/`}
  language="bash"
/>

Transfer both the Ollama binary and model archive to the air-gapped machine:

<CodeBlock
  code={`# On air-gapped machine
sudo cp ollama /usr/local/bin/
tar xzf ollama-models.tar.gz -C ~/

# Start Ollama
ollama serve`}
  language="bash"
/>

### Configure grith to use Ollama

<ConfigExample
  title="~/.config/grith/config.toml"
  code={`[llm]
default_provider = "ollama"

[llm.ollama]
base_url = "http://localhost:11434"
model = "llama3.1:8b"`}
/>

Verify connectivity between grith and Ollama:

<Terminal command='grith run "hello world"' />

<Callout type="tip" title="Model selection for air-gapped use">
For coding tasks, `codellama:13b` or `deepseek-coder:6.7b` provide good results on modest hardware. For general tasks, `llama3.1:8b` is a solid baseline. Larger models yield better results but require more RAM and GPU memory.
</Callout>

## Manual licence activation

Enterprise licences are normally validated online. In air-gapped environments, use offline activation.

### Generate a machine fingerprint

On the air-gapped machine, generate a hardware fingerprint:

<Terminal command="grith pro activate --offline --generate-fingerprint" />

<CodeBlock
  code={`[grith] Machine fingerprint generated:

    grith-fp-a1b2c3d4e5f6...

[grith] Send this fingerprint to your account manager
        or enter it at grith.ai/enterprise/activate`}
  language="text"
/>

### Obtain the offline licence

Transfer the fingerprint to a connected machine and submit it at [grith.ai/enterprise/activate](https://grith.ai/enterprise/activate), or send it to your grith account manager. You will receive a signed licence file (`grith-license.jwt`).

### Apply the licence

Transfer the licence file to the air-gapped machine and apply it:

<Terminal command="grith pro activate --offline --license-file grith-license.jwt" />

<CodeBlock
  code={`[grith] ✓ Offline licence activated
[grith] ✓ Tier: Enterprise
[grith] ✓ Valid until: 2027-02-24
[grith] ✓ Licensed to: Acme Corp (air-gapped deployment)`}
  language="text"
/>

<Callout type="info" title="Licence renewal">
Offline licences have a fixed expiry. Plan to renew before expiry by repeating the fingerprint/licence exchange process. grith continues operating with full functionality on an expired licence for a 30-day grace period, logging a warning at startup.
</Callout>

## Audit export for external compliance

In connected environments, audit data can be synced to the team dashboard. In air-gapped environments, export audit logs for transfer to external compliance systems.

### Export audit logs

<Terminal command="grith audit export --format json --output audit-2026-02.json" />

<Terminal command="grith audit export --format csv --output audit-2026-02.csv" />

### Export with compliance framing

For specific compliance frameworks, use the compliance export commands:

<CodeBlock
  code={`# SOC 2 evidence package
grith audit export --format json --compliance soc2 --output soc2-evidence.json

# NIST AI RMF measurement report
grith audit export --format json --compliance nist-ai-rmf --output nist-report.json

# FedRAMP ConMon package
grith audit export --format oscal --compliance fedramp --output fedramp-conmon.oscal.json`}
  language="bash"
/>

### Transfer schedule

Establish a regular cadence for transferring audit exports out of the air-gapped environment:

| Environment | Recommended frequency | Format |
|---|---|---|
| SCIF / classified | Weekly | Encrypted JSON via cross-domain solution |
| Regulated industry | Monthly | CSV or OSCAL for compliance tooling |
| Isolated dev lab | As needed | JSON for SIEM import |

<Callout type="warning" title="Audit log integrity">
grith uses hash-chained audit logs for tamper evidence. When transferring exports, preserve the chain verification metadata included in the export. Compliance tools can verify chain integrity independently.
</Callout>

## Update management

Without internet access, updates must be transferred manually.

### Update procedure

1. Download the new release on a connected machine
2. Verify the release signature (grith releases are signed with the project GPG key)
3. Transfer the binary via approved media
4. Stop any running grith sessions: `grith supervisor list` and `grith supervisor kill <id>`
5. Replace the binary: `sudo cp grith /usr/local/bin/`
6. Verify: `grith --version`
7. Update Ollama models if needed (same transfer process)

### Version pinning

In environments with strict change control, pin to a specific version and update only during approved maintenance windows. grith does not auto-update, so the installed version remains stable until explicitly replaced.

<ConfigExample
  title="~/.config/grith/config.toml"
  code={`[general]
# Log a warning if the running version does not match the expected version.
# Useful for change-control compliance in air-gapped environments.
expected_version = "1.0.0"`}
/>

## Network isolation verification

After setup, verify that grith operates correctly with no network access:

<CodeBlock
  code={`# Verify grith starts without network
grith --version

# Verify the proxy evaluates tool calls
grith proxy test fs.write ./test.txt "hello"

# Verify the built-in agent works with local LLM
grith run "create a hello world script"

# Verify supervisor mode works
grith exec --profile generic -- python -c "print('supervised')"

# Verify audit logging
grith audit`}
  language="bash"
/>

All of these commands should complete successfully with zero external network calls.
