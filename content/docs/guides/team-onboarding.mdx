## Overview

grith Pro includes team management: shared security policies, centrally managed provider API keys, and synchronised configuration across all team members. This guide walks through the setup process from first login to a fully onboarded team.

## Prerequisites

- A grith Pro subscription (or trial) with admin access at [grith.ai](https://grith.ai)
- grith v1.6+ installed on each team member's machine
- At least one LLM provider API key to distribute

## Step 1: Log in to grith Pro

The team admin authenticates from the CLI. This links the local grith installation to your grith.ai account.

<Terminal command="grith pro login" />

This opens a browser-based OAuth flow. If you are running in a headless environment or CI, use an API key instead:

<Terminal command="grith pro login --api-key YOUR_API_KEY" />

<CodeBlock
  code={`[grith] Authenticated as dan@example.com
[grith] Organisation: Acme Corp
[grith] Role: admin`}
  language="text"
/>

## Step 2: Activate your license

After login, activate the Pro license on this machine. This fetches your organisation's entitlements and stores them locally.

<Terminal command="grith pro activate" />

<CodeBlock
  code={`[grith] License activated
[grith] Tier: Pro
[grith] Seats: 12/25 used
[grith] Expires: 2027-03-24
[grith] Features: team-sync, encrypted-keys, policy-push, priority-support`}
  language="text"
/>

## Step 3: Add provider API keys on the dashboard

Navigate to the grith.ai dashboard and add your team's LLM provider API keys. Keys are encrypted at rest and distributed to team members via the sync mechanism.

Go to **Settings > Provider Keys** on [grith.ai](https://grith.ai/dashboard/settings/keys) and add keys for each provider your team uses (OpenAI, Anthropic, OpenRouter, etc.).

<Callout type="info" title="Key encryption">
Provider keys are encrypted with your organisation's key before storage. They are decrypted only on the team member's local machine after sync. grith.ai servers never see plaintext keys.
</Callout>

## Step 4: Create team policies on the dashboard

Team policies define shared security rules that apply to every team member. Navigate to **Settings > Policies** on the grith.ai dashboard.

Policies you can configure:

| Policy | Effect |
|---|---|
| Minimum deny threshold | Prevent team members from lowering the deny score below a floor |
| Required filters | Ensure specific filters (e.g., `secret_scan`, `egress_policy`) are always enabled |
| Denied destinations | Organisation-wide blocked network destinations |
| Denied paths | Organisation-wide blocked filesystem paths |
| Audit retention | Minimum local audit log retention period |

Policies are pushed to team members during sync and enforced locally. Team members cannot override organisation policies — they can only add stricter rules on top.

## Step 5: Onboard each team member

Each team member runs the same login and sync flow:

<Terminal command="grith pro login" />

After authenticating, they pull the team configuration:

<Terminal command="grith pro sync" />

<CodeBlock
  code={`[grith] Syncing with grith.ai...
[grith] ✓ Team policies updated (3 policies)
[grith] ✓ Provider keys synced (2 providers)
[grith] ✓ Supervisor profiles synced (1 custom profile)
[grith] Sync complete.`}
  language="text"
/>

This downloads and applies:
- Team security policies
- Encrypted provider API keys (decrypted locally)
- Any custom supervisor profiles shared by the admin

## Step 6: Verify the setup

Each team member can verify their Pro status and team configuration:

<Terminal command="grith pro status" />

<CodeBlock
  code={`grith Pro status:

  Account:      alice@example.com
  Organisation: Acme Corp
  Tier:         Pro
  Role:         member
  Last sync:    2026-03-24 10:15:03 UTC

  Provider keys:
    anthropic     ✓ (synced, encrypted)

## Step 7: Verify supervised execution

Confirm that a supervised session picks up the synced provider key and team policies:

<Terminal command='grith exec -- claude-code "list files in the current directory"' />

<CodeBlock
  code={`[grith] Profile: claude-code
[grith] Provider key: anthropic (team-synced)
[grith] Team policies: 3 active
[grith] Starting supervised session...`}
  language="text"
/>

The synced provider key is used automatically. Team members do not need to configure provider keys locally.

## Sync frequency

<Callout type="info" title="Sync schedule">
Team configuration syncs in two ways: manually via `grith pro sync`, and automatically in the background every 24 hours when the grith daemon is running. Provider keys are refreshed on the same schedule. Run `grith pro sync` after making changes on the dashboard to apply them immediately.
</Callout>

## Revoking access

To remove a team member, deactivate their seat on the grith.ai dashboard under **Settings > Team > Members**. Their local license will expire at the next sync check (within 24 hours). They can also be force-revoked immediately — the next `grith pro status` or any `grith exec` invocation will fail with an authentication error.
