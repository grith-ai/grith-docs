## Configuration file

grith stores its configuration in TOML format at `~/.config/grith/config.toml`. This file is created when you run `grith init` and contains all settings for the proxy, filters, LLM providers, digest, and server.

<Terminal command="grith init" />

## Viewing configuration

To view the current configuration and where each value comes from:

<Terminal command="grith config" />

<CodeBlock
  code={`[general]
  log_level = "info"                    # default
  data_dir = "~/.local/share/grith"     # default

[llm]
  default_provider = "ollama"           # config.toml

[proxy]
  auto_allow_threshold = 3.0            # config.toml
  auto_deny_threshold = 8.0             # config.toml
  cold_start_calls = 200                # default
  filter_chain = "default"              # default

[adaptive]

[digest]
  interval_active = "30m"               # default
  interval_idle = "24h"                 # default
  delivery = "cli"                      # default
  max_queue_size = 100                  # default

[server]
  dashboard_port = 3141                 # default

[supervisor]
  profile = "generic"                   # default

[notifications]
  enabled = false                       # default`}
  language="toml"
/>

## Modifying values

Use `grith config set` to modify individual values without editing the file by hand:

<Terminal command="grith config set proxy.auto_allow_threshold 3.0" />

<Terminal command="grith config set llm.default_provider openai" />

<Terminal command="grith config set general.log_level debug" />

Changes take effect on the next `grith run` or `grith exec` invocation. The dashboard picks up changes on restart.

## Configuration sections

### [general]

Top-level settings for logging and data storage.

<ConfigExample
  title="General settings"
  code={`[general]
log_level = "info"              # trace | debug | info | warn | error
data_dir = "~/.local/share/grith"
color = true                    # coloured terminal output`}
/>

### [llm]

LLM provider configuration for grith's built-in agent. See [LLM Providers](/docs/getting-started/providers) for full details on each provider.

<ConfigExample
  title="LLM settings"
  code={`[llm]
default_provider = "ollama"

[llm.ollama]
base_url = "http://localhost:11434"
model = "llama3.1:8b"

[llm.openai]
api_key_env = "OPENAI_API_KEY"
model = "gpt-4o-mini"

[llm.anthropic]
api_key_env = "ANTHROPIC_API_KEY"
model = "claude-sonnet-4-5-20250514"`}
/>

### [proxy]

Controls the security proxy's scoring thresholds and filter chain.

<ConfigExample
  title="Proxy settings"
  code={`[proxy]
auto_allow_threshold = 3.0      # below this score → ALLOW
auto_deny_threshold = 8.0       # above this score → DENY
cold_start_calls = 200          # calls before adaptive learning kicks in
filter_chain = "default"        # default | strict | permissive | custom
timeout_ms = 5000               # max time per filter evaluation
parallel_filters = true         # run independent filters concurrently`}
/>

<Callout type="info" title="Score ranges">
Tool calls scoring between `auto_allow_threshold` and `auto_deny_threshold` are queued for human review. Adjust these thresholds based on your risk tolerance.
</Callout>

### [adaptive]

Controls adaptive learning behaviour. grith observes patterns in tool calls over time and adjusts filter sensitivity accordingly.

<ConfigExample
  title="Adaptive settings"
  code={`[adaptive]
enabled = true                  # enable adaptive learning
cold_start_calls = 200          # calls before adaptive scores are used`}
/>

### [digest]

Controls the digest system for periodic summaries of session activity, queued items, and review notifications.

<ConfigExample
  title="Digest settings"
  code={`[digest]
interval_active = "30m"         # digest interval during active sessions
interval_idle = "24h"           # digest interval when idle
delivery = "cli"                # cli | email | webhook
max_queue_size = 100            # maximum queued items before forced review`}
/>

### [server]

Dashboard and API server configuration. Everything runs on a single port.

<ConfigExample
  title="Server settings"
  code={`[server]
dashboard_port = 3141           # web dashboard and API (single port)
bind_address = "127.0.0.1"     # localhost only by default`}
/>

### [supervisor]

Controls the supervisor agent that monitors the primary agent's behaviour.

<ConfigExample
  title="Supervisor settings"
  code={`[supervisor]
profile = "generic"             # generic | strict | permissive
enabled = true
review_interval_ms = 1000       # how often the supervisor checks state`}
/>

### [notifications]

Optional notifications for queued or denied tool calls.

<ConfigExample
  title="Notification settings"
  code={`[notifications]
enabled = true
backend = "desktop"             # desktop | slack | webhook

[notifications.slack]
webhook_url_env = "GRITH_SLACK_WEBHOOK"
channel = "#grith-alerts"

[notifications.webhook]
url = "https://your-endpoint.example.com/grith"
secret_env = "GRITH_WEBHOOK_SECRET"`}
/>

## Per-project overrides

Place a `.grith/config.toml` file in any project root to override global settings for that project. Per-project values take precedence over the global config.

<CodeBlock
  code={`# ~/projects/my-app/.grith/config.toml
# Stricter thresholds for this project

[proxy]
auto_allow_threshold = 1.5
auto_deny_threshold = 6.0
filter_chain = "strict"

[supervisor]
profile = "strict"`}
  language="toml"
/>

When you run `grith run` or `grith exec` from within that project directory, grith merges the per-project config on top of the global config. The merge order is:

1. Built-in defaults
2. Global config (`~/.config/grith/config.toml`)
3. Per-project config (`.grith/config.toml`)
4. Environment variable overrides

## Environment variable overrides

Any configuration value can be overridden via environment variables. The naming convention is `GRITH_` followed by the section and key in uppercase, separated by underscores.

| Variable | Overrides | Example |
|---|---|---|
| `GRITH_LOG_LEVEL` | `general.log_level` | `GRITH_LOG_LEVEL=debug` |
| `GRITH_CONFIG` | Config file path | `GRITH_CONFIG=/path/to/config.toml` |
| `GRITH_PROXY_AUTO_ALLOW_THRESHOLD` | `proxy.auto_allow_threshold` | `GRITH_PROXY_AUTO_ALLOW_THRESHOLD=3.0` |
| `GRITH_PROXY_AUTO_DENY_THRESHOLD` | `proxy.auto_deny_threshold` | `GRITH_PROXY_AUTO_DENY_THRESHOLD=7.0` |
| `GRITH_LLM_DEFAULT_PROVIDER` | `llm.default_provider` | `GRITH_LLM_DEFAULT_PROVIDER=openai` |
| `GRITH_SERVER_DASHBOARD_PORT` | `server.dashboard_port` | `GRITH_SERVER_DASHBOARD_PORT=8080` |

<Callout type="tip" title="CI environments">
Environment variables are particularly useful in CI pipelines where you want to override settings without modifying config files. Set `GRITH_CONFIG` to point to a CI-specific config, or override individual values.
</Callout>

## Minimal config example

A minimal configuration that uses Ollama with default settings:

<ConfigExample
  title="Minimal configuration"
  code={`[llm]
default_provider = "ollama"

[llm.ollama]
model = "llama3.1:8b"`}
/>

Everything else uses built-in defaults. This is what `grith init` generates.

## Full production config example

A complete configuration for a team environment with stricter security, notifications, and custom thresholds:

<ConfigExample
  title="Production configuration"
  code={`[general]
log_level = "info"
data_dir = "~/.local/share/grith"
color = true

[llm]
default_provider = "anthropic"

[llm.anthropic]
api_key_env = "ANTHROPIC_API_KEY"
model = "claude-sonnet-4-5-20250514"

[llm.openai]
api_key_env = "OPENAI_API_KEY"
model = "gpt-4o-mini"

[proxy]
auto_allow_threshold = 2.0
auto_deny_threshold = 7.0
cold_start_calls = 200
filter_chain = "strict"
timeout_ms = 5000
parallel_filters = true

[adaptive]
enabled = true

[digest]
interval_active = "30m"
interval_idle = "24h"
delivery = "cli"
max_queue_size = 100

[server]
dashboard_port = 3141
bind_address = "127.0.0.1"

[supervisor]
profile = "strict"
enabled = true
review_interval_ms = 500

[notifications]
enabled = true
backend = "slack"

[notifications.slack]
webhook_url_env = "GRITH_SLACK_WEBHOOK"
channel = "#grith-alerts"`}
/>
