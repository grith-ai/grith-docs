## System requirements

grith runs on macOS (Apple Silicon and Intel), Linux (x86_64 and aarch64), and Windows (x86_64). Full syscall interception is available on Linux via ptrace and seccomp-BPF. macOS operates in process-lifecycle fallback mode. Windows support is on the roadmap.

| Platform | Interception level | Status |
|---|---|---|
| Linux x86_64 / aarch64 | Full syscall (ptrace + seccomp-BPF) | Supported |
| macOS (Apple Silicon / Intel) | Process lifecycle fallback | Supported |
| Windows x86_64 | Planned (Minifilter + ETW) | Roadmap |

## Install via shell script

The fastest way to install grith on macOS or Linux:

<Terminal command="curl -fsSL https://grith.ai/install | sh" />

The installer detects your platform and architecture, downloads the appropriate binary, and places it in `/usr/local/bin`.

After installation, verify it works:

<Terminal command="grith --version" />

## Install via Homebrew

On macOS and Linux, grith is available as a Homebrew formula:

<Terminal command="brew install grith-ai/tap/grith" />

## Install on Windows

### winget

<Terminal command="winget install grith-ai.grith" />

### Scoop

<Terminal command="scoop bucket add grith https://github.com/grith-ai/scoop-bucket && scoop install grith" />

### MSI installer

Download the latest `.msi` from the [GitHub releases page](https://github.com/grith-ai/grith/releases).

## Build from source

If you prefer to build from source, you need a Rust toolchain (1.75+):

<Terminal command="git clone https://github.com/grith-ai/grith && cd grith && cargo build --release" />

The binary will be at `target/release/grith`. Copy it to a directory on your `$PATH`.

## Initial setup

After installing, initialise your configuration:

<Terminal command="grith init" />

This creates a default configuration file at `~/.config/grith/config.toml`. See [Configuration](/docs/getting-started/configuration) for details on customising it.
