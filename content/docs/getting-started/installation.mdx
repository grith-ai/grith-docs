## System requirements

grith is supported for production use on Linux today. macOS and Windows support is coming very soon.

| Platform | Status | Interception level |
|---|---|---|
| Linux x86_64 / aarch64 | Supported today | ptrace + seccomp-BPF |
| macOS (Apple Silicon / Intel) | Coming very soon | Planned platform release |
| Windows x86_64 | Coming very soon | Planned platform release |

<Callout type="info" title="Current launch scope">
The current launch targets Linux only. This page will gain macOS and Windows install instructions as soon as those builds ship.
</Callout>

## Install via shell script

The fastest way to install grith on Linux:

<Terminal command="curl -fsSL https://grith.ai/install | sh" />

The installer detects your Linux architecture, downloads the appropriate binary, and places it in `/usr/local/bin`.

After installation, verify it works:

<Terminal command="grith --version" />

## Install via Homebrew on Linux

If you already use Homebrew or Linuxbrew on Linux, grith is also available as a formula:

<Terminal command="brew install grith-ai/tap/grith" />

## Build from source

If you prefer to build from source on Linux, you need a Rust toolchain (1.80+):

<Terminal command="git clone https://github.com/grith-ai/grith && cd grith && cargo build --release" />

The binary will be at `target/release/grith`. Copy it to a directory on your `$PATH`.

## Initial setup

After installing, initialise your configuration:

<Terminal command="grith init" />

This creates a default configuration file at `~/.config/grith/config.toml`. See [Configuration](/docs/getting-started/configuration) for details on customising it.

<Callout type="tip" title="macOS and Windows">
If you are waiting on macOS or Windows support, check back here soon. The docs and install flows will be updated as part of those platform launches.
</Callout>
