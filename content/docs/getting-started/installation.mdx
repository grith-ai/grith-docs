## System requirements

grith runs on macOS (Apple Silicon and Intel) and Linux (x86_64 and aarch64). Full syscall interception is available on Linux via ptrace and seccomp-BPF. macOS operates in process-lifecycle fallback mode. Windows support is planned.

| Platform | Status | Interception level |
|---|---|---|
| Linux x86_64 / aarch64 | Supported | ptrace + seccomp-BPF |
| macOS (Apple Silicon / Intel) | Supported | Process lifecycle fallback |
| Windows x86_64 | Planned | Minifilter + ETW |

## Install via shell script

The fastest way to install grith on macOS or Linux:

<Terminal command="curl -fsSL https://grith.ai/install | sh" />

The installer detects your platform and architecture, downloads the appropriate binary, and places it in `/usr/local/bin`.

After installation, verify it works:

<Terminal command="grith --version" />

## Install via Homebrew

On macOS and Linux, grith is available as a Homebrew formula:

<Terminal command="brew install grith-ai/tap/grith" />

## Build from source

If you prefer to build from source, you need a Rust toolchain (1.80+):

<Terminal command="git clone https://github.com/grith-ai/grith && cd grith && cargo build --release" />

The binary will be at `target/release/grith`. Copy it to a directory on your `$PATH`.

## Initial setup

After installing, initialise your configuration:

<Terminal command="grith init" />

This creates a default configuration file at `~/.config/grith/config.toml`. See [Configuration](/docs/getting-started/configuration) for details on customising it.
